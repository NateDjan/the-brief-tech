Depuis son lancement en début d'année 2026, **OpenAI Codex** s'est imposé comme l'outil de développement IA le plus discuté dans les équipes tech. Mais OpenAI ne compte pas en rester là : au cours des deux premières semaines de juin 2026, la société a déployé une série de mises à jour majeures qui transforment Codex d'un assistant de code en une **plateforme de travail généraliste pour l'entreprise**.

Des plugins métiers spécialisés, une fonctionnalité pour créer et déployer des applications web en quelques secondes, et un mode développeur Chrome avancé : voici ce qui change.

## Les plugins métiers : Codex pour chaque rôle dans l'entreprise

La mise à jour la plus structurante est sans doute le déploiement des **plugins rôle-spécifiques dans Codex**. Disponibles pour les workspaces ChatGPT Enterprise, ces plugins packagent des compétences, intégrations d'applications, prompts de démarrage et guidances de workflow adaptés à des fonctions précises.

Le premier ensemble de **six plugins métiers** couvre :

- **Sales** — Préparation de prospects, recherche de comptes, aide à la rédaction de pitchs personnalisés, intégration CRM
- **Data Analytics** — Création de dashboards, analyses statistiques, visualisations automatiques depuis des sources de données d'entreprise
- **Creative Production** — Génération et itération d'assets créatifs pour le marketing et la communication
- **Product Design** — Prototypage rapide, user stories, documentation produit, wireframes textuels
- **Investment Banking** — Analyse financière, modélisation, due diligence, structuration de deals
- **Public Equity Investing** — Recherche boursière, analyse de rapports financiers, screening d'entreprises cotées

L'idée centrale est de sortir Codex du monde exclusif des développeurs pour en faire un **outil de travail quotidien** pour n'importe quel professionnel. Un analyste financier peut désormais demander à Codex de dépouiller un rapport annuel et le modéliser, sans avoir besoin d'écrire une ligne de code.

En complément, OpenAI a lancé **66 plugins d'applications uniques** supplémentaires qui élargissent les intégrations disponibles. Parmi les nouvelles connexions notables : **Databricks**, **Salesforce**, **Hex** (analytics collaboratif), et **Clay** (enrichissement de données commerciales). Ce catalogue positionne Codex comme une alternative sérieuse aux suites d'outils traditionnelles comme Microsoft 365 Copilot ou Google Workspace.

## ChatGPT Sites : créer et déployer une application web en langage naturel

La deuxième grande annonce concerne **ChatGPT Sites**, disponible en preview pour les workspaces Enterprise et Edu. Cette fonctionnalité représente un saut qualitatif significatif : **Codex peut désormais créer, itérer et déployer des applications web complètes** à partir d'une description en langage naturel.

Concrètement, les utilisateurs peuvent demander à Codex de créer une application JavaScript/TypeScript légère, qui sera hébergée avec une URL dédiée. Les applications Sites incluent :

- **Authentification "Sign in with ChatGPT"** — les accès peuvent être limités aux membres du workspace
- **Stockage de données et fichiers** — les apps peuvent persister des informations entre sessions
- **Partage workspace-interne** — les apps restent privées à l'organisation par défaut

Les cas d'usage immédiats sont nombreux : un outil de reporting interne, un tableau de bord de suivi de projet, une application de collecte de feedback client, un portail de documentation interactif. Tout cela sans recourir à une équipe de développement, en quelques minutes de conversation avec Codex.

Cette fonctionnalité positionne OpenAI directement en compétition avec des outils comme **Replit**, **Glide**, et dans une certaine mesure **Notion** pour les outils internes d'entreprise. La différence clé : pas d'interface de no-code à apprendre, juste du langage naturel.

## Developer Mode : Chrome DevTools dans Codex

Pour les développeurs qui utilisent déjà Codex, la mise à jour du 8 juin apporte une nouveauté très attendue : le **Developer Mode pour l'utilisation du navigateur Chrome**.

Quand activé depuis les paramètres de l'app Codex, ce mode donne à l'agent un accès contrôlé au **Chrome DevTools Protocol**, lui permettant de :

- **Profiler le JavaScript** pour identifier les goulots d'étranglement de performance
- **Inspecter les sorties de la console** et le trafic réseau en temps réel
- **Examiner l'état de la page** : DOM complet, styles CSS appliqués, structure du document
- **Diagnostiquer des problèmes en conditions réelles** sur des applications web en production

Le mode est désactivé par défaut — un choix délibéré de sécurité, puisque l'accès au DevTools Protocol permet théoriquement des interactions profondes avec le navigateur. Les utilisateurs doivent l'activer manuellement.

Pour les équipes de développement qui utilisent Codex au quotidien, c'est un changement de paradigme : l'agent peut désormais **déboguer des problèmes front-end directement dans le navigateur**, sans que le développeur ait besoin d'ouvrir lui-même les DevTools, de copier-coller des erreurs de console, ou de décrire manuellement ce qu'il voit à l'écran.

## Le contexte : vers un Codex "super-collègue"

Ces mises à jour s'inscrivent dans une vision claire d'OpenAI : transformer Codex en un **collègue IA universel** capable d'assister n'importe quel rôle dans une organisation moderne. La logique est cohérente avec les annonces du début de l'année autour des agents autonomes.

En quelques mois, Codex est passé de :
- Un assistant de code (génération, complétion, débogage)
- À un agent de développement (capable de travailler de façon autonome sur des tâches complexes)
- À une plateforme de travail d'entreprise (plugins métiers, création d'apps, intégrations)

Cette évolution n'est pas sans concurrents. **Microsoft Copilot** intègre des capacités similaires directement dans Teams, Word et Excel. **Google Gemini for Workspace** joue sur la même carte. **Anthropic** développe ses propres outils agentiques pour l'entreprise.

Mais OpenAI maintient un avantage important : la **base d'utilisateurs ChatGPT Enterprise** déjà installée dans des milliers d'organisations, et la vitesse à laquelle elle itère sur ses produits. Le rythme des mises à jour de juin 2026 est impressionnant — plusieurs fonctionnalités majeures en moins de deux semaines.

## Partage de plugins et collaboration

Une dernière fonctionnalité mérite d'être soulignée : le **partage de plugins dans les workspaces Enterprise**. Les utilisateurs peuvent désormais créer leurs propres plugins locaux (connecteurs vers des outils internes, workflows sur-mesure) et les partager avec leur équipe via un annuaire de plugins workspace.

C'est une capacité qui transforme Codex en **plateforme extensible** : les équipes les plus avancées peuvent construire leurs propres intégrations et les rendre disponibles à toute l'organisation sans intervention du support OpenAI. Un peu à la manière des "apps" dans Slack ou des "extensions" dans VS Code, mais alimentées par de l'IA agentique.

- **OpenAI Codex** reçoit des **plugins métiers spécialisés** pour 6 rôles : ventes, analytics, design produit, production créative, banque d'investissement et marchés actions
- **66 nouvelles intégrations applicatives** sont ajoutées (Databricks, Salesforce, Hex, Clay, etc.)
- **ChatGPT Sites** (preview Enterprise) permet de créer et déployer des apps web complètes en JavaScript/TypeScript via langage naturel
- **Developer Mode Chrome** donne à Codex un accès aux DevTools Protocol pour le débogage avancé de front-end
- **Partage de plugins workspace** : les équipes peuvent créer et distribuer leurs propres plugins internes
- Ces mises à jour transforment Codex d'un assistant de code en une **plateforme de travail IA généraliste** pour l'entreprise
- OpenAI accélère le rythme : plusieurs fonctionnalités majeures déployées en moins de deux semaines en juin 2026
